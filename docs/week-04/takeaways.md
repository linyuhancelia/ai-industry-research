## 本周核心认知清单

- [x] GPU 的本质：大量简单计算单元并行处理相同指令的"超级工厂"，不是"更强的 CPU"
- [x] SM（Streaming Multiprocessor）是 GPU 的基本工作单元——H100 有 132 个 SM，每个包含 128 个 CUDA Core + 4 个 Tensor Core
- [x] Tensor Core 一次完成 4×4 矩阵乘加（64 次 FMA），速度是 CUDA Core 的 4-16 倍
- [x] Warp（32 线程一组）是 GPU 最小执行单位——SIMT 模式一条指令驱动 32 个线程
- [x] 内存层次：寄存器→共享内存→L2 缓存→HBM，越近越快越小。HBM 是 GPU 的命门
- [x] Memory Wall（内存墙）：计算速度增长远快于内存带宽增长，内存常是真正瓶颈
- [x] HBM 仅 SK Hynix 和 Samsung 两家量产——双寡头垄断，定价权极强
- [x] Blackwell 三大设计决策：双芯封装突破光罩极限（2080 亿晶体管）、FP4 精度、NVLink 5.0
- [x] 理论算力 vs 实际算力：MFU 通常 30-50%，同样硬件优化水平不同可差 2.5 倍
- [x] 三大性能损耗：通信开销（20-30%）> 内存带宽瓶颈（15-25%）> 流水线气泡（10-15%）
- [x] 流水线气泡是 Pipeline Parallelism 的调度问题，与 CUDA Core/Tensor Core 无关
- [x] 通信开销优化：计算-通信重叠（Overlap），让 GPU 一边发送梯度一边计算下一批
- [x] 内存优化：先做免费的软件优化（FlashAttention、Kernel Fusion），再考虑换硬件
- [x] NVIDIA 护城河：硬件领先 2-4 年可追赶，CUDA 生态领先 10+ 年难替代。飞轮效应
- [x] NVIDIA GPU 毛利率 75-85%，定价权来自系统级切换成本而非硬件独占
- [x] AMD 的最优破局策略：B+C 组合——短期专攻推理市场（缝隙战略），长期推动开放生态联盟
- [x] FP4 是可用精度的实际极限（4 bit = 约 16 个离散值）
- [x] 推理成本降低扩大 TAM——不是零和博弈，是在做大整个市场

## 关键数字速查

| 数字 | 含义 |
|------|------|
| 132 个 SM | H100 的流式多处理器数量 |
| 128 + 4 | 每个 SM 中的 CUDA Core + Tensor Core 数量 |
| 32 | 一个 Warp 包含的线程数 |
| 3.35 TB/s → 8.0 TB/s | H100 → B200 的 HBM 带宽演进 |
| 80 GB → 192 GB | H100 → B200 的 HBM 容量演进 |
| 2,080 亿 | B200 双芯封装的晶体管总数 |
| 858 mm² | 光罩极限（单芯片最大面积） |
| 30-50% | 典型训练任务的 MFU 范围 |
| 75-85% | NVIDIA GPU 毛利率 |
| ~80% | NVIDIA 在 AI 训练 GPU 市场的份额 |
| 400 块 | 1,000 块 H100 在 MFU 40% 下的等价有效算力 |

## 形成的判断框架

### 框架 7：缝隙战略——面对飞轮型垄断的进攻路径
面对转动中的飞轮（如 NVIDIA 的 CUDA 生态），正面硬刚几乎不可能成功。有效策略是找到飞轮覆盖不到的缝隙（如推理市场——CUDA 依赖弱、性价比敏感），在缝隙中建立根据地，再逐步扩大。诊断不是处方——分析清楚问题后，必须给出具体的战略行动方案。

### 框架 8：先免费后付费的优化序列
面对性能/效率问题时，先穷尽免费的软件优化（通信重叠、算子融合、FlashAttention），再考虑花钱的硬件升级（换 HBM、换 GPU）。类比策略运营：先把免费流量玩法做到极致，再投预算买付费流量。

### 框架 9：推理成本与 TAM 的正反馈
推理成本降低不是零和博弈（省钱），而是在扩大市场（做大蛋糕）。成本降一半 → 更多场景的 Unit Economics 跑通 → 更多企业上线 AI → 推理总调用量暴增 → GPU 需求不降反增。评估技术进步的商业影响时，要看它打开的增量市场，而不只是存量市场的成本节省。
