## 本周核心认知清单

- [x] Google TPU 不是"更好的 GPU"，而是从头设计的 AI 专用芯片（ASIC）
- [x] TPU 的核心优势是垂直整合：芯片+互联+网络+编译器+框架+模型全部自研
- [x] Google 做 TPU 的根本动机是供应链自主权 + 长期成本优势，不是"做更好的芯片"
- [x] Google 不卖 TPU 是因为"卖服务"比"卖产品"更赚钱——更高毛利、经常性收入、客户锁定
- [x] AMD MI300X 硬件参数超过 H100，但实际训练 MFU 低 10-20 个百分点（ROCm 差距）
- [x] AMD 的最优突破口是推理市场——HBM 大（192GB）+ 价格低 + CUDA 依赖弱
- [x] 华为昇腾在中国市场的竞争对手是 H20（阉割版），不是 H100——这个对比下华为有竞争力
- [x] 出口管制是华为昇腾市场需求的结构性驱动力——管制创造市场边界
- [x] ASIC（专用芯片）在特定场景性能极致，但缺乏生态和通用性
- [x] GPU vs ASIC 终局：训练市场 GPU 主导，推理市场 GPU+ASIC 共存，边缘推理 ASIC 主导
- [x] 推理市场竞争远比训练激烈——CUDA 飞轮在推理场景转得更慢
- [x] "双轨策略"是地理维度的分轨（海外训练 + 国内推理），不是时间维度
- [x] ASIC 创业公司破局的关键是"单点极致"——选 1-2 个模型做到极致，找合作伙伴免费 POC
- [x] 硅晶圆是单晶硅材料，纯度 11 个 9；芯片缺陷来自制造工艺而非材料本身
- [x] Cerebras 用冗余核心设计（多造 5-10% 备用核心）解决整片晶圆的良率挑战
- [x] 战略分析的完整链路 = 判断（做什么）+ 路径（怎么做）+ 风险（可能出什么错）
- [x] 答题时先确认"题目问的是谁的决策、从谁的视角分析"，避免偏离核心问题

## 形成的判断框架

### 框架 10：垂直整合 vs 水平分工（Vertical Integration vs Horizontal Specialization）

**决策变量**：什么时候应该像 Google 一样自研全栈，什么时候应该像大多数公司一样外购 NVIDIA？

| 条件 | 垂直整合（自研） | 水平分工（外购） |
|------|----------------|----------------|
| **规模** | 规模巨大，足以摊薄自研成本 | 规模不够大，自研 ROI 为负 |
| **供应链风险** | 供应商锁定是战略威胁 | 供应商竞争充分，风险可控 |
| **组织能力** | 有全栈工程能力 | 缺乏硬件/底层软件团队 |
| **典型案例** | Google TPU、Apple 自研芯片 | 大多数 AI 创业公司用 NVIDIA |

**跟 Week 3 模型 5 的联系**：组织能力决定技术选型——垂直整合只有顶级组织能做到。

### 框架 11："卖产品" vs "卖服务"的商业模型选择

| 维度 | 卖产品（一次性销售） | 卖服务（经常性收入） |
|------|--------------------|--------------------|
| **毛利率** | 40-60% | 60-70% |
| **收入模式** | 一锤子买卖 | 按月/按量持续付费 |
| **客户锁定** | 弱——买完可以换 | 强——数据/代码/工作流迁移成本高 |
| **市场规模** | 更广（任何人可买） | 较窄（需要基础设施支撑） |
| **估值倍数** | 较低（P/E） | 较高（P/S，资本市场偏爱经常性收入） |

**应用场景**：不只适用于芯片 vs 云服务。任何"卖工具 vs 卖解决方案"的商业模式选择都可以用这个框架。

### 框架 12：管制驱动的市场结构（Regulation-Shaped Markets）

出口管制、政策法规可以**人为创造市场边界**，改变竞争的参照系。

**关键洞察**：在"围墙花园"市场内，竞争对手和竞争力评估都不同于全球市场。华为昇腾在全球市场打不过 H100，但在中国市场（出口管制下）它的竞争对手是 H20——这个对比下华为有明显优势。

**应用场景**：分析任何受政策影响的市场时，先画出"围墙"在哪里，再在围墙内部做竞争分析。中国 AI 芯片、欧盟数据市场（GDPR）、医药市场（FDA 审批）都适用。
