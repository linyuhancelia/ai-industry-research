## 本周核心认知清单

- [ ] 内存墙的物理根源：GPU 算力每代增长 2-3 倍，HBM 带宽仅增长 1.5-2.4 倍
- [ ] Roofline 模型：低算术强度的操作被内存带宽"压死"，GPU 算力再高也没用
- [ ] AI 的大量辅助操作（LayerNorm、Softmax、KV Cache）都是内存受限，拖累整体 MFU
- [ ] 内存金字塔：SRAM（最快最贵最小）→ HBM → DDR → SSD → 网络存储（最慢最便宜最大）
- [ ] SRAM 成本是 HBM 的 50-100 倍——Cerebras 的 44GB SRAM 方案极贵
- [ ] 数据搬运消耗的能量是计算本身的 10-100 倍
- [ ] HBM 占 GPU 物料成本的 40-60%——SK Hynix 拿走了 GPU 产业链近一半的成本
- [ ] HBM 供应链双寡头：SK Hynix（53%）+ Samsung（38%）
- [ ] CoWoS 先进封装是 GPU 出货的最紧瓶颈——不是 GPU 芯片不够，是封装不够
- [ ] KV Cache 是推理场景的内存墙核心——上下文越长，内存压力越大
- [ ] CXL 不能替代 HBM（带宽差 10 倍），但能作为 HBM 的补充（KV Cache 溢出、内存池化）
- [ ] 打破内存墙三条路线：搬近（HBM 堆叠）、搬到（存内计算 PIM）、少搬（量化/稀疏化）
