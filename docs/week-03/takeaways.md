## 本周核心认知清单

- [x] 网络是 AI 训练的"隐形瓶颈"——每步通信延迟 × 百万步 = 天级成本差异（$1,130 万案例）
- [x] 三层互联架构：芯片级（NVLink C2C）→ 机内（NVLink+NVSwitch）→ 跨机（IB/以太网），越近越快
- [x] NVLink 5.0 的质变：72 GPU NVLink Domain，张量并行不再受限于单机
- [x] InfiniBand vs 以太网的本质是"垄断 vs 开放"的商业模式之争
- [x] UEC（Ultra Ethernet Consortium，超以太网联盟）目标：开发 AI 专用以太网标准，打破 NVIDIA 网络垄断
- [x] Fat-Tree 拓扑实现无阻塞（Non-blocking），但交换机数量巨大（16K GPU 需 800-1,000 台）
- [x] Rail-Optimized 拓扑匹配并行通信模式，网络成本降 20-30%，性能几乎无损
- [x] 光模块占网络成本 40-60%，是 AI 基础设施的隐形成本大头
- [x] 三种并行策略（DP/TP/PP）的通信模式与网络层匹配：TP→NVLink，PP→IB/NVLink Domain，DP→IB/以太网
- [x] NVIDIA 三层锁定中，CUDA（软件生态）最难替代，InfiniBand（跨机网络）最容易被突破
- [x] 组织能力决定技术选型——同一技术对 Meta 是最优解，对创业公司可能是灾难
- [x] 光模块的价值捕获模式是"周期性溢价 + 趋势性增长"，非持续超额利润也非简单量大利薄

## 形成的判断框架

### 框架 4：软件生态锁定 > 硬件锁定
硬件锁定可以"买新设备"打破（贵但一次性），软件生态锁定是累积效应——开发者数量、库生态、项目存量每天都在增加切换成本。评估护城河持久度时，软件生态层的壁垒始终高于硬件层。

### 框架 5：组织能力决定技术选型
同一个技术方案，对不同规模的组织可能是最优解或灾难。分析技术方案时，除了"技术可行性"，还要问"我的团队能驾驭吗？"。Meta 用以太网替代 IB 成功，是因为有 1,000+ 网络工程师；创业公司做同样选择可能导致训练集群长期不稳定。

### 框架 6：技术代际窗口期
在技术换代期（如 400G→800G 光模块），早期具备新一代产品能力的少数厂商享受 18-24 个月的溢价窗口。产能追上需求后回归竞争均衡。投资时机（Timing）比选公司更重要。

## 与 Week 4 的衔接

第一阶段讲清楚了"GPU 在什么物理环境中运行"——电力供给、散热系统、网络连接。Week 4 开始进入第二阶段：**芯片与硬件层**——这些 GPU 本身是怎么设计的？

- NVIDIA 的护城河到底是硬件（GPU 架构）还是软件（CUDA 生态）？Week 3 已经建立了"CUDA 最难替代"的判断，Week 4 会从硬件架构角度验证

- GPU 的"算力"数字（如 "1 petaFLOPS"）到底意味着什么？为什么实际训练效率远低于理论峰值？
